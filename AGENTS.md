# Инструкции для агентов

## Контекст

`YT Lang Learning` — Electron-приложение без фреймворка UI. Главный процесс находится в `main.js`, интерфейс — в `app.js` и `app.css`. Связь между ними выполняется только через API из `preload.js`.

## Команды

```powershell
bun run start
bun run test
bun run package
bun run make
```

Перед завершением изменений запускайте `bun run test`, если они затрагивают JavaScript-логику. Для изменений упаковки используйте также `bun run package` или `bun run make` по необходимости.

## Упаковка и диагностика

- После `bun run package` проверяйте `out/yt-lang-learning-win32-x64/resources/app.asar`: все локальные скрипты из `index.html` должны присутствовать в архиве. Например: `& .\node_modules\.bin\asar.exe list .\out\yt-lang-learning-win32-x64\resources\app.asar`.
- В `packagerConfig.ignore` используйте регулярные выражения для сегментов пути. Запрещены широкие шаблоны вроде `/out/`: они могут исключить обычный файл по части его имени, например `player-layout.js`.
- Если упакованное окно пустое или плеер не загружается, сначала получите точный текст ошибки renderer/main-процесса и проверьте состав `app.asar`. Не выпускайте несколько сборок подряд, меняя неподтверждённые гипотезы.

## Правила изменений

- Не подключайте зависимости и фреймворки без явной необходимости.
- Не предоставляйте интерфейсу прямой доступ к Node.js или Electron: расширяйте `preload.js` и обработчики IPC в `main.js`.
- Новую изолированную логику выносите в отдельный модуль и добавляйте к нему тест на `node:test` по образцу существующих `*.test.js`.
- Не создавайте новые тесты без явного запроса пользователя. Существующие тесты не удаляйте; исправлять их можно только если это необходимо из-за изменений в коде или если они перестали корректно проверять поведение.
- Сохраняйте обратную совместимость структур `library.json`, `settings.json` и `captions/*.json`; при изменении схемы добавляйте нормализацию или миграцию.
- Не записывайте API-ключи и локальные абсолютные пути в документацию, тесты или исходный код. Ключ перевода хранится в настройках в зашифрованном виде.
- Не редактируйте `out/`, `node_modules/` и `.bun-tmp/`: это генерируемые каталоги.
- Не удаляйте и не перезаписывайте пользовательскую библиотеку, настройки или субтитры без явного действия пользователя.

## Данные

В разработке `library.json`, `settings.json` и каталог `captions/` расположены в корне проекта. В упакованной версии приложение ищет их рядом с исполняемым файлом.

## Стиль

- Используйте существующий стиль JavaScript: CommonJS в main/preload и обычный браузерный JavaScript в UI.
- Тексты интерфейса — на русском.
- Делайте изменения минимальными и проверяемыми; не смешивайте рефакторинг с функциональной задачей без необходимости.
