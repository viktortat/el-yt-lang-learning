# Технічна документація YT Lang Learning

YT Lang Learning — локальний Electron-застосунок для перегляду YouTube-відео з субтитрами мови, яку вивчаєте, та перекладом. Інтерфейс — на звичайному браузерному JavaScript без UI-фреймворку.

![Навчальний плеєр YT Lang Learning](screens/study-player.png)

Інструкції для користувачів — у [посібнику](guide-users.md). Кореневі сторінки проекту: [English README](../README.md) та [український README](../README.uk.md).

## Вимоги

- Windows для інсталятора Squirrel та описаної збірки;
- Bun для встановлення залежностей і запуску команд проекту;
- інтернет для YouTube, отримання доступних субтитрів і перекладу через OpenRouter;
- `uv`, `yt-dlp` і модель faster-whisper для локальної транскрибації (необовязково).

Версії Bun, `uv`, `yt-dlp` і faster-whisper у проекті не закріплені. Перевіряйте сумісність у своєму середовищі перед випуском збірки.

## Запуск із вихідного коду

```powershell
bun install
bun run start
```

Точка входу головного процесу вказана в `package.json` і веде на `main.js`.

## Перевірки

```powershell
bun run test
```

Команда запускає тести на `node:test` для синхронізації субтитрів, розбору доріжок, керування плеєром, розкладки, збереження позиції, плану транскрибації та зв'язків між файлами застосунку.

## Збірка та пакування

```powershell
bun run package
bun run make
```

`bun run package` створює розпакований застосунок у `out/yt-lang-learning-win32-x64/`. `bun run make` створює інсталятор `out/make/squirrel.windows/x64/yt-lang-learning-setup.exe` та ZIP-архів. Каталог `out/` генерується заново, не редагуйте його вручну.

## Компоненти

- `main.js` створює вікно Electron, читає та записує дані, працює з YouTube, запускає транскрибацію та звертається до OpenRouter.
- `preload.js` виставляє обмежений API через `contextBridge`.
- `app.js` керує бібліотекою, плеєром, субтитрами та налаштуваннями.
- `app.css` містить стилі інтерфейсу.
- `caption-parser.js`, `caption-sync.js`, `player-controls.js`, `player-layout.js`, `playback-position.js`, `transcription-plan.js` та `translation-response.js` містять окрему перевірену логіку.

Вікно запускається з `contextIsolation: true` та `nodeIntegration: false`. Інтерфейс не отримує прямого доступу до Node.js або Electron.

## Дані

У режимі розробки застосунок читає та записує:

```text
library.json
settings.json
captions/<video-id>.json
```

У корені проекту — в розробці, у `%LOCALAPPDATA%/YT Lang Learning` — у запакованій версії. Не видаляйте та не перезаписуйте їх під час оновлення користувацької установки.

Chromium та користувацькі дані застосунку використовують стабільну папку `userData`: `%LOCALAPPDATA%/YT Lang Learning`.

Структури нормалізуються під час читання. Зміна схеми потребує зворотної сумісності або міграції.

## Налаштування та зовнішні сервіси

`settings.json` зберігає тему, назву моделі OpenRouter та параметри локальної транскрибації. API-ключ шифрується через `safeStorage`. Якщо системне шифрування недоступне, застосунок не дасть зберегти новий ключ.

Переклад надсилає репліки мовою оригіналу в `https://openrouter.ai/api/v1/chat/completions` пакетами по 25 елементів. Отриманий текст зберігається у відповідному файлі `captions/*.json`.

Субтитри завантажуються через `yt-dlp`. Якщо готової доріжки немає, інтерфейс окремо запитує дозвіл на тимчасове завантаження відео та локальний запуск faster-whisper. Тимчасові каталоги видаляються після завершення.

## Обмеження

- Метадані та відтворення залежать від YouTube і правил вбудовування конкретного ролика.
- Доріжка може бути відсутньою або непридатною для завантаження.
- OpenRouter потребує робочий API-ключ і доступ до вказаної моделі.
- Шляхи локальної транскрибації за замовчуванням заточені на поточне Windows-середовище. Перед поширенням їх потрібно налаштувати.

## Правила змін

- Зв'язок із головним процесом розширюйте через `preload.js` та IPC-обробники в `main.js`.
- Не додавайте прямий доступ інтерфейсу до Node.js або Electron.
- Нову ізольовану логіку супроводжуйте тестом на `node:test`.
- Не записуйте API-ключі та локальні абсолютні шляхи у вихідний код, тести й документацію.
- Не редагуйте `out/`, `node_modules/` та `.bun-tmp/`.
