# Техническая документация YT Lang Learning

YT Lang Learning — локальное Electron-приложение для просмотра YouTube-видео с английскими субтитрами и русским переводом. Интерфейс — на обычном браузерном JavaScript без UI-фреймворка.

Пользовательские инструкции — в [руководстве](guide-users.md). Корневые страницы проекта: [English README](../README.md) и [русский README](../README.ru.md).

## Требования

- Windows для установщика Squirrel и описанной сборки;
- Bun для установки зависимостей и запуска команд проекта;
- интернет для YouTube, получения доступных субтитров и перевода через OpenRouter;
- `uv`, `yt-dlp` и модель faster-whisper для локальной транскрибации (необязательно).

Версии Bun, `uv`, `yt-dlp` и faster-whisper в проекте не закреплены. Проверяйте совместимость в своей среде перед выпуском сборки.

## Запуск из исходников

```powershell
bun install
bun run start
```

Точка входа главного процесса указана в `package.json` и ведёт на `main.js`.

## Проверки

```powershell
bun run test
```

Команда запускает тесты на `node:test` для синхронизации субтитров, разбора дорожек, управления плеером, раскладки, сохранения позиции, плана транскрибации и связей между файлами приложения.

## Сборка и упаковка

```powershell
bun run package
bun run make
```

`bun run package` создаёт распакованное приложение в `out/yt-lang-learning-win32-x64/`. `bun run make` создаёт установщик `out/make/squirrel.windows/x64/yt-lang-learning-setup.exe` и ZIP-архив. Каталог `out/` генерируется заново, не редактируйте его вручную.

## Компоненты

- `main.js` создаёт окно Electron, читает и записывает данные, работает с YouTube, запускает транскрибацию и обращается к OpenRouter.
- `preload.js` выставляет ограниченный API через `contextBridge`.
- `app.js` управляет библиотекой, плеером, субтитрами и настройками.
- `app.css` содержит стили интерфейса.
- `caption-parser.js`, `caption-sync.js`, `player-controls.js`, `player-layout.js`, `playback-position.js`, `transcription-plan.js` и `translation-response.js` содержат отдельную проверяемую логику.

Окно запускается с `contextIsolation: true` и `nodeIntegration: false`. Интерфейс не получает прямой доступ к Node.js или Electron.

## Данные

В режиме разработки приложение читает и записывает:

```text
library.json
settings.json
captions/<video-id>.json
```

В корне проекта — в разработке, в `%LOCALAPPDATA%/YT Lang Learning` — в упакованной версии. Не удаляйте и не перезаписывайте их во время обновления пользовательской установки.

Chromium и пользовательские данные приложения используют стабильную папку `userData`: `%LOCALAPPDATA%/YT Lang Learning`.

Структуры нормализуются при чтении. Изменение схемы требует обратной совместимости или миграции.

## Настройки и внешние сервисы

`settings.json` хранит тему, имя модели OpenRouter и параметры локальной транскрибации. API-ключ шифруется через `safeStorage`. Если системное шифрование недоступно, приложение не даст сохранить новый ключ.

Перевод отправляет английские реплики в `https://openrouter.ai/api/v1/chat/completions` пакетами по 25 элементов. Полученный русский текст сохраняется в соответствующем файле `captions/*.json`.

Английские субтитры загружаются через `yt-dlp`. Если готовой дорожки нет, интерфейс отдельно спрашивает разрешение на временную загрузку видео и локальный запуск faster-whisper. Временные каталоги удаляются после завершения.

## Ограничения

- Метаданные и воспроизведение зависят от YouTube и правил встраивания конкретного ролика.
- Английская дорожка может отсутствовать или оказаться непригодной для загрузки.
- OpenRouter требует рабочий API-ключ и доступ к указанной модели.
- Пути локальной транскрибации по умолчанию заточены на текущую Windows-среду. Перед распространением их нужно настроить.

## Правила изменений

- Связь с главным процессом расширяйте через `preload.js` и IPC-обработчики в `main.js`.
- Не добавляйте прямой доступ интерфейса к Node.js или Electron.
- Новую изолированную логику сопровождайте тестом на `node:test`.
- Не записывайте API-ключи и локальные абсолютные пути в исходный код, тесты и документацию.
- Не редактируйте `out/`, `node_modules/` и `.bun-tmp/`.
