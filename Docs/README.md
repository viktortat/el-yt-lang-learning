# Техническая документация YT Lang Learning

YT Lang Learning - локальное Electron-приложение для просмотра YouTube-видео с английскими субтитрами и русским переводом. Интерфейс написан на обычном браузерном JavaScript без UI-фреймворка.

Пользовательские инструкции находятся в [руководстве](guide-users.md). Корневые страницы проекта: [English README](../README.md) и [русский README](../README.ru.md).

## Требования

- Windows для установщика Squirrel и описанного процесса упаковки;
- Bun для установки зависимостей и запуска команд проекта;
- интернет для YouTube, получения доступных субтитров и перевода через OpenRouter;
- `uv`, `yt-dlp` и скачанная модель faster-whisper для необязательной локальной транскрибации.

Версии Bun, `uv`, `yt-dlp` и faster-whisper в проекте не закреплены. Проверяйте совместимость в своей среде перед выпуском сборки.

## Запуск из исходников

```powershell
bun install
bun run start
```

Точка входа главного процесса указана в `package.json` и ведёт на `main.js`.

## Проверки

```powershell
bun run test
```

Команда запускает тесты на `node:test` для синхронизации субтитров, разбора дорожек, управления плеером, раскладки, сохранения позиции, плана транскрибации и связей между файлами приложения.

## Сборка и упаковка

```powershell
bun run package
bun run make
```

`bun run package` создаёт распакованное приложение в `out/yt-lang-learning-win32-x64/`. `bun run make` создаёт установщик `out/make/squirrel.windows/x64/yt-lang-learning-setup.exe` и ZIP-архив. Каталог `out/` генерируется заново и не должен редактироваться вручную.

## Компоненты

- `main.js` создаёт окно Electron, читает и записывает данные, работает с YouTube, запускает транскрибацию и обращается к OpenRouter.
- `preload.js` публикует ограниченный API через `contextBridge`.
- `app.js` управляет библиотекой, плеером, субтитрами и настройками.
- `app.css` содержит стили интерфейса.
- `caption-parser.js`, `caption-sync.js`, `player-controls.js`, `player-layout.js`, `playback-position.js` и `transcription-plan.js` содержат отдельную проверяемую логику.

Окно запускается с `contextIsolation: true` и `nodeIntegration: false`. Интерфейс не получает прямой доступ к Node.js или Electron.

## Данные

В режиме разработки приложение читает и записывает:

```text
library.json
settings.json
captions/<video-id>.json
```

Эти файлы находятся в корне проекта. В упакованной версии они находятся рядом с исполняемым файлом. Не удаляйте и не перезаписывайте их во время обновления пользовательской установки.

Chromium использует отдельный каталог `userData` в `%LOCALAPPDATA%/YT Lang Learning`. Он не заменяет библиотеку и файлы субтитров рядом с приложением.

Структуры `library.json`, `settings.json` и `captions/*.json` нормализуются при чтении. Изменение их схемы требует сохранения обратной совместимости или миграции.

## Настройки и внешние сервисы

`settings.json` хранит тему, имя модели OpenRouter и параметры локальной транскрибации. API-ключ OpenRouter шифруется через Electron `safeStorage`. Если системное шифрование Windows недоступно, приложение отклоняет сохранение нового ключа.

Перевод отправляет английские реплики в `https://openrouter.ai/api/v1/chat/completions` пакетами по 50 элементов. Полученный русский текст сохраняется в соответствующем файле `captions/*.json`.

Загрузка английских субтитров использует `yt-dlp`. Если готовой дорожки нет, интерфейс отдельно спрашивает разрешение на временную загрузку видео и локальный запуск faster-whisper. Временные каталоги удаляются в блоке завершения операции.

## Ограничения

- Метаданные и воспроизведение зависят от YouTube и правил встраивания конкретного ролика.
- Английская дорожка может отсутствовать или оказаться непригодной для загрузки.
- OpenRouter требует рабочий API-ключ и доступ к указанной модели.
- Значения путей локальной транскрибации по умолчанию ориентированы на текущую Windows-среду и должны настраиваться перед распространением приложения.

## Правила изменений

- Расширяйте связь с главным процессом через `preload.js` и обработчики IPC в `main.js`.
- Не добавляйте прямой доступ интерфейса к Node.js или Electron.
- Новую изолированную логику сопровождайте тестом `node:test`.
- Не записывайте API-ключи и локальные абсолютные пути в исходный код, тесты и документацию.
- Не редактируйте `out/`, `node_modules/` и `.bun-tmp/`.
